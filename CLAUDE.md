You can read the files in the `specs` directory for high-level context about what we are building.

For an overview of the contents of the prototype, see CODE_OVERVIEW.md in this folder. You can then use this to explore the parts of the codebase that are relevant; I suggest starting with the specs/overview.md and then reading the plans relevant to your task.

Whenever you make a substantial change to the codebase organization (e.g. adding a new file, or refactoring the fundamental structure of something), please also modify CODE_OVERVIEW.md so that it is up to date! (You do not need to write a chronological log of everything you changed here, this doc should instead describe the _current_ state of the codebase, for someone coming to it with fresh eyes. You should make sure that the file listing is still accurate, that any new fundamental concepts are briefly mentioned, and that generally if someone was asked to make a change that they could use this file to figure out what to read and what to change.)

Plans go in `plans` (in the project directory, not the ~/.claude directory). When writing plans, treat the plan like a high-level spec for how to implement the given task. You can ask clarification questions if you have them, using either your ask-questions tool or free-form questions over chat, and then finally you can write a consise plan about how to implement the feature. When writing a plan, code blocks should be small, and should focus on explaining the moving parts, interface boundaries, design decisions, and what would need to be done to implement it rather than including actual snippets of the implementation. Pseudocode is OK. Write it as if you expect someone new to read it to understand what you are planning to do, or to refer back to it during code review once you have a real implementation.

When creating a plan, you may want to think through the details first. To avoid cluttering the plan, you should create a new file `{name_of_plan}_scratch.md` adjacent to the original plan doc, and write your scratch thoughts there.
